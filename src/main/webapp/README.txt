Чтобы упростить создание файла архива веб-приложения в требуемом формате, удобно расположить «исполняемые» файлы вашего
 веб-приложения (то есть файлы, которые Tomcat фактически использует при выполнении вашего приложения) в одной организации,
 как того требует сам формат WAR. Для этого в корневом каталоге документов вашего приложения появится следующее содержимое:

* .html, * .jsp и т. д. - страницы HTML и JSP вместе с другими файлами, которые должны быть видны клиентскому браузеру
(например, JavaScript, файлы таблиц стилей и изображения) для вашего приложения. В больших приложениях вы можете разделить
эти файлы на иерархию подкаталогов, но для небольших приложений, как правило, гораздо проще поддерживать только один каталог для этих файлов.

/WEB-INF/web.xml - дескриптор развертывания веб-приложения для вашего приложения. Это XML-файл, описывающий сервлеты и другие
компоненты, составляющие ваше приложение, вместе с любыми параметрами инициализации и ограничениями безопасности, управляемыми
контейнером, которые вы хотите, чтобы сервер установил для вас. Этот файл обсуждается более подробно в следующем подразделе.

/ WEB-INF / classes / - этот каталог содержит любые файлы классов Java (и связанные ресурсы), необходимые для вашего приложения,
включая классы сервлетов и не сервлетов, которые не объединены в файлы JAR. Если ваши классы организованы в пакеты Java, вы должны
отразить это в иерархии каталогов в / WEB-INF / classes /. Например, класс Java с именем com.mycompany.mypackage.MyServlet
необходимо сохранить в файле с именем /WEB-INF/classes/com/mycompany/mypackage/MyServlet.class.

/ WEB-INF / lib / - этот каталог содержит файлы JAR, которые содержат файлы классов Java (и связанные ресурсы), необходимые
для вашего приложения, такие как сторонние библиотеки классов или драйверы JDBC.
Когда вы устанавливаете приложение в Tomcat (или любой другой контейнер сервлетов 2.2 или более поздней версии),
классы в каталоге WEB-INF / classes /, а также все классы в файлах JAR, найденных в каталоге WEB-INF / lib /, становятся
становится видимым для других классов в вашем конкретном веб-приложении. Таким образом, если вы включите все необходимые
классы библиотек в одно из этих мест (не забудьте проверить лицензии на право распространения для любых сторонних библиотек,
которые вы используете), вы упростите установку своего веб-приложения - никаких настроек системы путь к классу (или установка
файлов глобальной библиотеки на вашем сервере) будет необходимо.

Большая часть этой информации была извлечена из главы 9 спецификации Servlet API версии 2.3, к которой вам следует обратиться
для получения более подробной информации.

--------------------------------------------------------------------------------------------------------------------------------

Как упоминалось выше, файл /WEB-INF/web.xml содержит дескриптор развертывания веб-приложения для вашего приложения.
Как следует из расширения имени файла, этот файл представляет собой XML-документ и определяет все, что нужно знать серверу
 о вашем приложении (за исключением пути контекста, который назначается системным администратором при развертывании приложения).

Полный синтаксис и семантика дескриптора развертывания определены в главе 13 спецификации Servlet API версии 2.3. Ожидается,
 что со временем появятся инструменты разработки, которые будут создавать и редактировать дескриптор развертывания за вас.
 А пока в качестве отправной точки предоставляется базовый файл web.xml. Этот файл включает комментарии, описывающие назначение
 каждого включенного элемента. (src/main/webapp/WEB_INF/XML.txt)

ПРИМЕЧАНИЕ. - Спецификация сервлета включает дескриптор типа документа (DTD) для дескриптора развертывания веб-приложения,
а Tomcat применяет определенные здесь правила при обработке файла /WEB-INF/web.xml вашего приложения. В частности, вы должны
 ввести элементы дескриптора (такие как <filter>, <servlet> и <servlet-mapping> в порядке, определенном DTD (см. Раздел 13.3).

 -------------------------------------------------------------------------------------------------------------------------------

 Файл /META-INF/context.xml можно использовать для определения конкретных параметров конфигурации Tomcat, таких как журнал доступа,
  источники данных, конфигурация диспетчера сеансов и многое другое. Этот XML-файл должен содержать один элемент Context, который
  будет считаться дочерним элементом элемента Host, соответствующего узлу, на котором развертывается веб-приложение. Документация
  по настройке Tomcat содержит информацию об элементе Context.
